### Функциональное программирование: краткое введение

**Что это?**  
Функциональное программирование (ФП) — это парадигма программирования, в которой вычисления трактуются как вычисление математических функций. Основной акцент делается на **чистые функции**, **неизменяемость данных** и **отсутствие побочных эффектов**.

**Ключевые принципы**  
1. **Чистые функции**: функция всегда возвращает один и тот же результат при одинаковых аргументах и не влияет на внешнее состояние (например, не изменяет глобальные переменные).  
2. **Неизменяемость**: данные после создания не меняются. Вместо изменения объекта создаётся новый.  
3. **Функции как объекты первого класса**: их можно передавать как аргументы, возвращать из других функций и присваивать переменным.

**Практический пример (на Python)**  
Представим задачу: возвести в квадрат все чётные числа из списка.

Императивный подход:
```python
numbers = [1, 2, 3, 4, 5]
result = []
for n in numbers:
    if n % 2 == 0:
        result.append(n ** 2)
```

Функциональный подход:
```python
numbers = [1, 2, 3, 4, 5]
result = list(map(lambda x: x ** 2, filter(lambda x: x % 2 == 0, numbers)))
```
Или ещё чище с генератором списков (в духе ФП):
```python
result = [x ** 2 for x in numbers if x % 2 == 0]
```

**Почему это полезно?**  
Код становится проще для тестирования, отладки и параллелизации, так как отсутствуют скрытые зависимости и изменяемое состояние. Функциональный стиль особенно эффективен при работе с потоками данных, обработке коллекций и в распределённых системах.
