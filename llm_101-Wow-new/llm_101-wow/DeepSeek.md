### **Что такое функциональное программирование (ФП)?**

Функциональное программирование — это **стиль (парадигма) разработки программ**, в котором программа строится на основе функций, понимаемых в математическом смысле. Код состоит из набора выражений и функций, которые преобразуют данные, а не из последовательности команд, меняющих состояние программы.

### **Ключевые принципы ФП**

1.  **Чистые функции**
    Это самый важный принцип. Чистая функция:
    *   **Всегда возвращает одинаковый результат для одних и тех же входных данных.** Это делает её поведение предсказуемым.
    *   **Не имеет побочных эффектов.** Она не изменяет внешние переменные, не модифицирует входные аргументы и не выполняет операций ввода-вывода (например, вывод в консоль или запрос к сети).

2.  **Неизменяемость данных (Immutability)**
    Данные никогда не изменяются "на месте". Вместо изменения существующей структуры создаётся её новая, изменённая копия. Это предотвращает неожиданные ошибки, когда несколько частей программы пытаются изменить одни и те же данные.

3.  **Функции высшего порядка**
    Это функции, которые могут принимать другие функции в качестве аргументов или возвращать их как результат. Это позволяет создавать гибкие и переиспользуемые абстракции.

### **Практический пример: Императивный vs Функциональный подход**

**Задача:** Увеличить зарплату всем сотрудникам в массиве на 10%.

**Императивный подход (как "делать"):**
Мы описываем *как* это сделать: "пройдись по массиву и для каждого элемента измени значение".

```javascript
let employees = [
  { name: 'Alice', salary: 50000 },
  { name: 'Bob', salary: 60000 }
];

for (let i = 0; i < employees.length; i++) {
  employees[i].salary = employees[i].salary * 1.1; // Изменяем исходный массив
}
```

**Функциональный подход (что "получить"):**
Мы описываем *что* мы хотим получить: "новый массив, где зарплата каждого сотрудника увеличена на 10%".

```javascript
const employees = [
  { name: 'Alice', salary: 50000 },
  { name: 'Bob', salary: 60000 }
];

const updatedEmployees = employees.map(employee => ({
  ...employee, // Создаём копию объекта сотрудника
  salary: employee.salary * 1.1 // И заменяем в ней поле salary
}));
// Исходный массив `employees` остался неизменным!
```

### **Преимущества ФП**

*   **Предсказуемость и надёжность:** Чистые функции и неизменяемость делают код менее подверженным ошибкам.
*   **Лёгкость тестирования:** Чистые функции легко тестировать, так как они не зависят от внешнего состояния.
*   **Параллелизм:** Без изменяемого состояния проще писать код для многопоточных систем.

ФП не всегда заменяет другие подходы, но его принципы (особенно чистые функции) полезно применять в любом проекте для повышения качества кода.
