### Функциональное программирование: базовое понимание

#### Что это такое

Функциональное программирование (ФП) — это парадигма, в которой программы строятся как **последовательность применения функций** к данным. В отличие от императивного стиля (где мы даём компьютеру пошаговые инструкции), в ФП мы описываем *что* нужно получить, а не *как* это сделать.

#### Ключевые принципы

1. **Неизменяемость данных** (*immutability*).  
   После создания данные нельзя изменить — вместо этого создаются новые значения. Это исключает побочные эффекты и упрощает отладку.

2. **Чистые функции** (*pure functions*).  
   Функция всегда возвращает один и тот же результат для одних и тех же аргументов и не влияет на внешнее состояние (нет `print`, изменения глобальных переменных и т. п.).

3. **Функции высшего порядка** (*higher-order functions*).  
   Функции могут принимать другие функции в качестве аргументов и/или возвращать их.

4. **Рекурсия** вместо циклов.  
   Повторение действий реализуется через вызов функции из неё же самой.

#### Почему это полезно

- **Проще тестировать**: чистые функции легко проверять — вход → выход.
- **Легче параллелить**: нет общего изменяемого состояния → меньше гонок (race conditions).
- **Выразительность**: часто код короче и понятнее.

#### Практический пример

Сравним решение одной задачи в императивном и функциональном стиле (на Python).

**Задача**: удвоить каждый элемент списка и оставить только чётные.

*Императивный стиль:*
```python
result = []
for x in [1, 2, 3, 4, 5]:
    x *= 2
    if x % 2 == 0:
        result.append(x)
```

*Функциональный стиль:*
```python
result = list(filter(lambda x: x % 2 == 0, map(lambda x: x * 2, [1, 2, 3, 4, 5])))
```

Здесь:
- `map` применяет функцию `$x \to 2x$` к каждому элементу.
- `filter` оставляет только чётные значения.
- Всё выражается как цепочка преобразований без изменяемых переменных.

#### Популярные языки ФП

- Haskell (чистое ФП)
- Scala, F# (гибридные)
- JavaScript, Python (поддерживают ФП-подход)

**Итог**: ФП — это мощный инструмент для написания надёжного, читаемого кода. Начинать можно даже в языках, где ФП не является основным стилем.
